# זכרון גורלי: קלוד קוד טוען וצורב זכרון חי `load` & `/burn/`

שתי פקודות סלאש שסוגרות את לולאת ההקשר (קונטקסט) ב-Claude Code:

<ul dir="rtl">
<li><b><code>/load</code></b> — בתחילת שיחה, שולף את כל מה שהפרויקט כבר יודע (הערות, מזהים, החלטות עבר, כללים שסיכמת עם קלוד).</li>
<li><b><code>/burn</code></b> — בסוף שיחה, סורק את מה שקרה בה וכותב בחזרה למקום הנכון: מה למדנו, אילו שגיאות נתקלנו בהן, ומה אמרת ל-Claude "מעכשיו תעשה ככה".</li>
</ul>

התוצאה: במקום להתחיל כל שיחה מהתחלה ולהסביר הכל שוב, הפרויקט שלך טוען ידע וקונטקסט אוטומטית. `/load` בתחילת שיחה הבאה כבר יודע את כל מה שלמדנו בפעם הקודמת.

---

## התקנה

פותחים שיחה חדשה ב-Claude Code ומדביקים את הבלוק הבא:

<div dir="ltr">

```
Install /load and /burn from https://raw.githubusercontent.com/arielmoatti/claude-code-recall-reflect/main/INSTALL.md — read the file and follow it exactly. Communicate with me in Hebrew throughout.
```

</div>

&rlm;Claude ימשוך את ה-INSTALL.md, יסרוק את המערכת הקיימת שלך (סקילים, זיכרון, תיקיות פרויקטים), וינהל אשף התקנה קצר בעברית שמתאים את הפקודות לסביבה שלך. בסוף התהליך יהיו לך:

<ul dir="rtl">
<li><code>~/.claude/commands/load.md</code> — פקודת <code>/load</code> מותאמת אישית</li>
<li><code>~/.claude/commands/burn.md</code> — פקודת <code>/burn</code> מותאמת אישית</li>
<li>תבנית <code>.claude/LOAD.md</code> בפרויקט הנוכחי (אם בחרת ליצור אותה)</li>
</ul>

ההתקנה לא נוגעת בפקודות קיימות עם אותו שם — אם כבר יש לך `load.md` או `burn.md`, האשף יציע לך "שמור כמו שזה" או "החלף עם גרסה חדשה" לפני כל כתיבה.

## איך זה עובד

### מחזור החיים של שיחה

<ol dir="rtl">
<li><b>פותחים שיחה חדשה</b> — מקלידים <code>/load</code> וזהו. Claude קורא את <code>.claude/LOAD.md</code> בפרויקט הנוכחי, שמפנה אותו לכל קבצי הזיכרון, ה-REGISTRY, ה-ARCH_PATTERNS וכל מה שחשוב.</li>
<li><b>עובדים</b> — בלי להסביר מחדש כל פעם מה זה הפרויקט הזה, איפה ה-IDs, איזה כללים יש.</li>
<li><b>בסוף השיחה</b> — מקלידים <code>/burn</code> (או Claude עצמו יציע). הפקודה סורקת את השיחה ומחלקת כל ידע חדש לאחד משלושה מקומות:</li>
</ol>

<ul dir="rtl">
<li><b>שגיאות ולקחים שליליים</b> — מה שלא עבד, לולאות שנתקענו בהן, הנחות שגויות ← <code>negative-knowledge.md</code></li>
<li><b>הוראות שנתת</b> — "אל תעשה X", "מעכשיו תמיד Y" ← קובצי <code>feedback_*.md</code> בזיכרון הפרויקט</li>
<li><b>התקדמות ו-IDs חדשים</b> ← <code>CURRENT_SPRINT.md</code>, <code>REGISTRY.md</code></li>
</ul>

### מבחן הזיהום

לפני שכותבים משהו, `/burn` שואל: "האם המידע הזה כולל IDs ספציפיים של הלקוח (מספרי תרחישים, טבלאות, webhooks)?"

<ul dir="rtl">
<li><b>כן</b> ← זיכרון פרויקט בלבד</li>
<li><b>לא — זה דפוס כללי</b> ← סקיל גלובלי ב-<code>~/.claude/skills/</code></li>
</ul>

זה מונע מידע של לקוח אחד לדלוף לפרויקטים אחרים.

## שאלות נפוצות

<b>מה קורה אם אין לי עדיין קבצי זיכרון בפרויקט?</b><br>
&rlm;האשף יציע ליצור תבנית <code>.claude/LOAD.md</code> ריקה בפרויקט הנוכחי, עם הערות על מה להוסיף בהמשך. <code>/load</code> יעבוד גם אם ה-LOAD.md רק מצביע על README.md.

<b>האם זה עובד רק בעברית?</b><br>
&rlm;הפקודות עצמן אגנוסטיות — Claude עונה בשפה שבה אתה כותב. האשף וההוראות בפנים כתובים בעברית כי זו שפת הקהילה שלנו.

<b>מה קורה אם אני רוצה לשנות את הקטלוג של <code>/burn</code> אחרי ההתקנה?</b><br>
&rlm;פשוט הריץ את בלוק ההתקנה שוב. האשף יזהה שכבר יש <code>burn.md</code> ויציע לערוך.

<b>למה הפקודה נקראת "burn"?</b><br>
&rlm;מתוך הרגל: בסוף שיחה טובה הייתי אומר ל-Claude "בא נסכם, תעבור על מה שעשינו, ותצרוב את הלקחים". "צריבה" כמו לצרוב לדיסק דיוידי — לקבע.
