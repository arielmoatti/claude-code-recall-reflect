# INSTALL — recall-reflect orchestrator

**מטרת הקובץ הזה:** מנהל את תהליך ההתקנה של `/load` ו-`/burn`. קורא: Claude (המודל שמבצע את ההתקנה עבור המשתמש).

## כלל על (אל תעבור עליו)

**כל התקשורת עם המשתמש — בעברית.** גם אם המשתמש כותב אליך מילה או שתיים באנגלית, תענה בעברית. טוקנים טכניים (שמות פקודות, נתיבי קבצים, קוד) נשארים באנגלית בתוך הטקסט העברי.

**אל תכתוב כלום לדיסק עד שהמשתמש אישר.** בכל שלב של שינוי קובץ — הצג לו את השינוי המתוכנן וקבל אישור מפורש.

## Phase 0 — פתיחה

הצג למשתמש פתיחה קצרה (2-3 משפטים) בעברית:

> ברוך הבא להתקנת `/load` ו-`/burn` — שתי פקודות שיחסכו לך חזרות בין שיחות. התהליך קצר: אני סורק את מערכת Claude הקיימת שלך, שואל אותך כמה שאלות, וכותב את הפקודות המותאמות אישית. מוכן להתחיל?

חכה לאישור ("כן", "קדימה", "yes", או כל אישור חיובי).

---

## Phase 1 — סריקת המערכת

לאחר אישור, סרוק את הסביבה הנוכחית. בצע את הפעולות הבאות במקביל כשאפשר:

### מה לסרוק

<!-- suppress dir="rtl"; this is Claude-facing -->

1. **`~/.claude/commands/`** — האם כבר קיים `load.md` או `burn.md`? אם כן, קרא אותם לזכרון כדי שנוכל להשוות בהמשך.
2. **`~/.claude/skills/`** — רשום את כל קובצי ה-`.md` שם. אלו הסקילים הגלובליים של המשתמש.
3. **`~/.claude/CLAUDE.md`** — קיים? אם כן, קרא אותו כדי להבין את סגנון העבודה של המשתמש והאם יש חוקים גלובליים רלוונטיים (למשל memory routing).
4. **תיקיית הפרויקט הנוכחי** (cwd) — בדוק:
   - האם יש `.claude/LOAD.md`? (אם כן, `/load` כבר פעיל בפרויקט הזה)
   - האם יש `CLAUDE.md`?
   - האם יש תיקיית `.claude/memory/`?
   - האם יש `README.md`?
5. **תיקיות פרויקטים נוספות** — **לא לסרוק** (מחוץ לסקופ ויכול להיות הרבה). אם המשתמש ירצה, הוא יזמין אותך לסריקה רחבה יותר בהמשך.

### מה לדווח

לאחר הסריקה, סכם למשתמש בעברית מה מצאת. פורמט מומלץ:

```
סיימתי סריקה. הנה מה שמצאתי:

• פקודות קיימות: [load.md: ישנה / אין] [burn.md: ישנה / אין]
• סקילים גלובליים: [מספר] קבצים — [שמות עיקריים או "תיקייה ריקה"]
• CLAUDE.md גלובלי: [קיים / אין]
• הפרויקט הנוכחי: [שם התיקייה]
  – LOAD.md: [קיים / אין]
  – memory/: [קיים / אין]
  – README.md: [קיים / אין]

ממשיך לחלק השאלות?
```

חכה לאישור.

## Phase 4 — סיכום סופי

הצג למשתמש סיכום בעברית של מה נוצר, מה שונה, ומה לא. פורמט:

```
ההתקנה הושלמה.

נכתב/עודכן:
  ✓ ~/.claude/commands/load.md
  ✓ ~/.claude/commands/burn.md
  ✓ <project>/.claude/LOAD.md  [אם רלוונטי]

דולג (לפי בקשתך):
  – [מה לא שונה ולמה]

בדיקה מהירה: פתח שיחה חדשה עם Claude Code בתיקיית הפרויקט הזה, והקלד /load. Claude אמור לקרוא את ה-LOAD.md ולדווח שהוא טעון.

להיכרות עם פקודת /burn: בסוף השיחה הזאת (או כל שיחה עתידית), הקלד /burn ותראה איך הוא עובד.
```

## הערות למבצע (Claude)

- **קריאה לפני כתיבה** — לפני שאתה כותב קובץ חדש, וודא שקראת את הקובץ הקיים (אם יש) כדי לאפשר השוואה.
- **אל תריץ `git push` או שום פעולה מרוחקת** — כל השינויים מקומיים.
- **אם סריקה נכשלת** (הרשאה חסרה, קובץ לא קיים) — ספר למשתמש בעברית בעדינות ושאל איך להמשיך. אל תמציא תוצאות.
- **אם המשתמש מתנגד באמצע** — עצור מיד, סכם מה כן ומה לא נעשה, ובקש הוראות.
